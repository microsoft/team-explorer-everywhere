Das Team Explorer Everywhere-Plug-In und der Befehlszeilenclient werden installiert

Sie können Team Explorer Everywhere und Visual Studio Team Foundation Server in Ihren Entwicklungsprozessen mit Eclipse verwenden, um Arbeitselemente nachzuverfolgen, Versionskontrolle bereitzustellen und Builds zu automatisieren. Sie müssen das Team Explorer Everywhere-Plug-In für Eclipse oder den plattformübergreifenden Befehlszeilenclient für Team Foundation Server installieren, bevor Sie diese Features verwenden können.

In diesem Thema

  • Erforderliche Komponenten
  • Team Explorer Everywhere-Plug-In für Eclipse installieren
  • Installieren des plattformübergreifenden Befehlszeilenclients für Team Foundation Server

Erforderliche Komponenten

Sie müssen eine der unterstützten Versionen von Eclipse installieren, die Sie auf der folgenden Website finden: Eclipse.org-Startseite.

Betriebssystemanforderungen:

  • Windows 10 (x86 und x64)
  • Windows 8.1 (x86 und x64)
  • Windows 8 (x86, x64)
  • Windows 7 (x86, x64)
  • Linux mit GLIBC 2.3+ (x86, x86_64)
  • Mac OS X 10.8+ (nur Intel)

Anforderungen an die integrierte Entwicklungsumgebung (IDE):

  • IDEs auf Grundlage von Eclipse 4.2+ unter Linux, Mac OS X oder Windows.

Java Runtime-Anforderungen:

  • Oracle Java™ 1.7+ oder IBM Java™ 1.7+ unter Linux
  • Apple Java™ 1.7+ unter Mac OS X
  • Oracle Java™ 1.7+ oder IBM Java™ 1.7+ unter Windows

Anforderungen an Team Foundation Server:

  • Visual Studio Team Services
  • Visual Studio Team Foundation Server 2017
  • Visual Studio Team Foundation Server 2015
  • Visual Studio Team Foundation Server 2013
  • Visual Studio Team Foundation Server 2012

Team Explorer Everywhere-Plug-In für Eclipse installieren

Sie müssen das Plug-In zunächst installieren, um es bei der Entwicklung einsetzen zu können. Sie können direkt von der Update-Site oder aus dem Eclipse-Update-Site-Archiv in Eclipse installieren.

So installieren Sie das Plug-In von der Update-Site

  1. Öffnen Sie Eclipse.
  2. Wählen Sie im Menü Hilfe den Befehl Neue Software installieren.
  3. Das Dialogfeld Installieren wird angezeigt.
  4. Klicken Sie auf Hinzufügen.
  5. Das Dialogfeld Repository hinzufügen wird angezeigt.
  6. Geben Sie als Namen "Team Explorer Everywhere" ein.
  7. Geben Sie den Standort der Update-Site an: http://dl.microsoft.com/eclipse/tfs
  8. Klicken Sie auf OK.
  9. Aktivieren Sie in der Liste der Features im Dialogfeld "Installieren" das Kontrollkästchen, das Team Explorer Everywhere-Plug-In für Eclipse entspricht. Wenn diese Option nicht angezeigt wird, verwenden Sie das Pulldownmenü für Arbeiten mit, und suchen Sie nach der Updatewebsite-URL, die Sie soeben in die Liste eingegeben haben. Wählen Sie diese aus, und aktivieren Sie dann das Kontrollkästchen neben dem oben genannten Plug-In.
  10. Klicken Sie zwei Mal auf Weiter.
  11. Wenn Sie die Microsoft Software-Lizenzbedingungen akzeptieren, stimmen Sie ihnen zu, und klicken Sie dann auf Fertig stellen.
  12. Starten Sie Eclipse neu, wenn Sie dazu aufgefordert werden.

So installieren Sie das Plug-In aus dem Update-Site-Archiv

    Hinweis
    Der obige Standardmechanismus für Update-Sites ist die bevorzugte Installationsmethode. Allerdings wird dafür eine Verbindung zum Internet benötigt, und es wird jeweils die neueste Version des Plug-Ins von Microsoft installiert. Wenn der Installationscomputer nicht direkt mit dem Internet verbunden ist oder von den Benutzern in der Organisation eine bestimmte Version des Plug-Ins installiert werden soll, ist die folgende Methode besser geeignet.

  1. Laden Sie über den Link auf dieser Seite "TFSEclipsePlugin-UpdateSiteArchive-14.0.4.zip" herunter.
  2. Öffnen Sie Eclipse.
  3. Wählen Sie im Menü Hilfe den Befehl Neue Software installieren.
  4. Das Dialogfeld Installieren wird angezeigt.
  5. Klicken Sie auf Hinzufügen.
  6. Das Dialogfeld Repository hinzufügen wird angezeigt.
  7. Geben Sie als Namen "Lokales Team Explorer Everywhere-Updatearchiv" ein.
  8. Klicken Sie auf Archiv.
  9. Das Dialogfeld Repository-Archiv wird angezeigt.
  10. Suchen Sie die heruntergeladene Datei "TFSEclipsePlugin-UpdateSiteArchive-14.0.4.zip", und wählen Sie sie als Dateinamen aus.
  11. Klicken Sie auf Öffnen.
  12. Klicken Sie auf OK.
  13. Aktivieren Sie in der Liste der Features im Dialogfeld "Installieren" das Kontrollkästchen, das Team Explorer Everywhere-Plug-In für Eclipse entspricht.
  14. Klicken Sie zwei Mal auf Weiter.
  15. Wenn Sie die Microsoft Software-Lizenzbedingungen akzeptieren, stimmen Sie ihnen zu, und klicken Sie dann auf Fertig stellen.
  16. Starten Sie Eclipse neu, wenn Sie dazu aufgefordert werden.

Installieren des plattformübergreifenden Befehlszeilenclients für Team Foundation Server

Mit dem plattformübergreifenden Befehlszeilenclient für Team Foundation Server können viele Vorgänge der Versionskontrolle ausgeführt werden. Um diesen Client verwenden zu können, müssen Sie ihn zuerst installieren und den Installationsordner in Ihren Shell- oder Systempfad aufnehmen.

So installieren Sie den plattformübergreifenden Befehlszeilenclient für Team Foundation Server

  1. Laden Sie über den Link auf dieser Seite "TEE-CLC-14.0.4.zip" herunter.
  2. Entpacken Sie die Archivdatei (TEE-CLC-14.0.4.zip), die den Client enthält.
  3. Nehmen Sie den Ordner, in den Sie das Archiv entpackt haben, in den Shell- oder Systempfad auf.
  4. Um zu überprüfen, ob der Client funktioniert, geben Sie an einer Eingabeaufforderung tf ein und drücken die EINGABETASTE.
Wenn der Client richtig installiert ist, wird eine Ausgabe beginnend mit "Microsoft Team Explorer Everywhere-Befehlszeilenclient (Version 14.0.4)" angezeigt.

Um Informationen zu einem Befehl zu erhalten, können Sie tf help Command eingeben. Dabei ist Command der Name des Befehls, zu dem Sie Informationen benötigen. Sie können zum Beispiel tf help checkin eingeben. Sie können auch tf command /help oder tf command -help angeben, um Hilfe abzurufen.

Hinweis
Durch Festlegen von Umgebungsvariablen in der Shell können Sie das Verhalten des Befehlszeilenclients steuern. Sie können zum Beispiel einen HTTP-Proxy angeben, die automatische Zwischenspeicherung Ihrer Anmeldeinformationen aktivieren oder das Tool für Unterschiede oder Zusammenführung steuern. Informationen zu den festzulegenden Umgebungsvariablen finden Sie im Thema Umgebungsvariablen festlegen auf MSDN.
